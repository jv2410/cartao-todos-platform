## Technical Notes

### Database Schema

#### users table
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  force_password_change BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
```

#### sessions table
```sql
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token VARCHAR(500) NOT NULL,
  expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
  device_type VARCHAR(50),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_token ON sessions(token);
CREATE INDEX idx_sessions_expires_at ON sessions(expires_at);
```

#### api_credentials table
```sql
CREATE TABLE api_credentials (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  waba_id VARCHAR(50) NOT NULL,
  phone_number_id VARCHAR(50) NOT NULL,
  encrypted_access_token TEXT NOT NULL,
  business_manager_id VARCHAR(50),
  last_tested_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_api_credentials_user_id ON api_credentials(user_id);
```

#### audit_logs table
```sql
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE SET NULL,
  action VARCHAR(100) NOT NULL,
  resource_type VARCHAR(100),
  resource_id VARCHAR(255),
  changes_json JSONB,
  ip_address VARCHAR(45),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_audit_logs_user_id ON audit_logs(user_id);
CREATE INDEX idx_audit_logs_action ON audit_logs(action);
CREATE INDEX idx_audit_logs_created_at ON audit_logs(created_at);
CREATE INDEX idx_audit_logs_resource_type ON audit_logs(resource_type);
```

### Migration Runner

#### Node.js Migration Script
- **File**: `backend/src/scripts/migrate.ts`
- **Purpose**: Execute SQL migration files in order, track executed migrations
- **Features**:
  - Read migration files from `backend/src/migrations/` directory
  - Execute migrations in alphanumeric order (001, 002, 003, 004)
  - Track executed migrations in `schema_migrations` table
  - Skip already-executed migrations (idempotent)
  - Rollback support (execute rollback SQL in reverse order)

#### Migrations Tracking Table
```sql
CREATE TABLE IF NOT EXISTS schema_migrations (
  version VARCHAR(255) PRIMARY KEY,
  executed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### Database Connection

#### PostgreSQL Configuration
- **File**: `backend/src/config/database.ts`
- **Connection**: Use `pg` (node-postgres) library
- **Connection String**: Loaded from environment variable `DATABASE_URL`
- **Format**: `postgresql://user:password@host:port/database?sslmode=require`
- **Connection Pool**: Max 20 connections, idle timeout 30 seconds

#### Environment Variables
```env
DATABASE_URL=postgresql://admin:password@localhost:5432/cartao_todos_dev
```

### Rollback Script

#### Rollback Migration
- **File**: `backend/src/scripts/rollback.ts`
- **Purpose**: Revert last migration or all migrations
- **Usage**:
  - `npm run migrate:rollback` - Rollback last migration
  - `npm run migrate:rollback:all` - Rollback all migrations (drop all tables)

## Notes
- This story should be implemented FIRST in Epic 1 (all other stories depend on database tables).
- Use `gen_random_uuid()` for UUID generation (PostgreSQL 13+ built-in function, no need for external libraries).
- Bcrypt cost factor 10 is a good balance between security and performance (~100ms per hash).
- Default admin password "100101" is intentionally simple for MVP (must be changed on first login via Story 1.2).
- CASCADE delete for sessions and credentials ensures no orphaned data when user is deleted.
- SET NULL for audit_logs ensures audit trail is preserved even if user is deleted (LGPD compliance).
- JSONB column for changes_json allows flexible audit log data structure (can store any JSON object).
- schema_migrations table tracks executed migrations (prevents re-running migrations).
- Migration runner should use transactions (rollback if any migration fails).
- Consider adding `updated_at` trigger for automatic timestamp updates (Phase 2 enhancement).
- Railway (hosting platform) supports PostgreSQL 15.x (use compatible syntax, avoid version-specific features).
