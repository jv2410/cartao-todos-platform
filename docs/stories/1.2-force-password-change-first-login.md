# Story 1.2: Force Password Change on First Login

## Status: Completed

## Description
As an admin, I want to be forced to change the default password on my first login, so that the platform is secured with a unique password only I know.

## Acceptance Criteria
- [x] Given I am a new user with default password "100101", when I successfully log in, then I am immediately redirected to /change-password page (cannot access dashboard)
- [x] Given I am on the change-password page, when I try to navigate away or access other routes, then I am redirected back to /change-password until password is changed
- [x] Given I am changing my password, when I enter the current password (default "100101"), then the field validates correctly
- [x] Given I enter a new password, when it does not meet complexity requirements, then I see inline error messages: "Minimum 8 characters", "Must include uppercase letter", "Must include lowercase letter", "Must include number", "Must include special character (!@#$%^&*)"
- [x] Given I enter a new password equal to the default password "100101", when I submit, then I see error "New password must be different from default password"
- [x] Given I enter a new password in the "New Password" field, when I enter a different password in "Confirm Password" field, then I see error "Passwords do not match"
- [x] Given I enter a valid new password matching all requirements, when I click "Change Password", then the password is updated in the database (bcrypt hashed)
- [x] Given my password change is successful, when the update completes, then I see success message "Password changed successfully" and am automatically logged in and redirected to /dashboard
- [x] Given my password has been changed, when I log in again later, then the force_password_change flag is false and I go directly to dashboard
- [x] Given I am on the change-password page, when I click "Logout" link, then I am logged out and redirected to /login page

## Technical Notes

### Database Updates
- **users** table update:
  - Update `password_hash` field with new bcrypt hash
  - Set `force_password_change = false` after successful password change
  - Update `updated_at` timestamp

### API Endpoints
- **POST /api/auth/change-password**
  - Request body: `{ current_password: string, new_password: string }`
  - Headers: `Authorization: Bearer {token}`
  - Response: `{ success: boolean, message: string }`
  - Status codes: 200 (success), 400 (validation error), 401 (invalid current password or unauthorized), 500 (server error)

- **GET /api/auth/me** (existing endpoint from Story 1.1):
  - Response includes `force_password_change` boolean flag
  - Used to determine if user should be redirected to change-password page

### Frontend Components
- **Change Password Page** (`/frontend/src/app/change-password/page.tsx`):
  - Form with three fields: Current Password, New Password, Confirm Password
  - Real-time validation feedback for password complexity
  - Password strength indicator (visual bar showing weak/medium/strong)
  - Submit button disabled until all validations pass
  - Success message and auto-redirect to dashboard
  - Logout link at bottom of form

- **Password Validation Component** (`/frontend/src/components/password-requirements.tsx`):
  - Displays password requirements checklist
  - Each requirement shows green checkmark when met, gray when not met
  - Requirements: min 8 chars, uppercase, lowercase, number, special char

### Password Complexity Validation
- **Minimum 8 characters**
- **At least one uppercase letter** (A-Z)
- **At least one lowercase letter** (a-z)
- **At least one number** (0-9)
- **At least one special character** (!@#$%^&*)
- **Cannot be the same as default password** ("100101")
- **New password and confirmation must match**

### Security Requirements
- Current password must be verified before allowing change (prevent unauthorized password changes)
- New password must be hashed with bcrypt (cost factor 10) before storing
- Password validation must occur on both frontend (UX) and backend (security)
- Audit log entry created when password is changed (user_id, action: "PASSWORD_CHANGED", timestamp, IP address)
- Session remains valid after password change (no need to log in again)

### Validation Logic (Backend)
```typescript
function validatePassword(password: string): { valid: boolean; errors: string[] } {
  const errors: string[] = [];

  if (password.length < 8) {
    errors.push("Password must be at least 8 characters");
  }
  if (!/[A-Z]/.test(password)) {
    errors.push("Password must include at least one uppercase letter");
  }
  if (!/[a-z]/.test(password)) {
    errors.push("Password must include at least one lowercase letter");
  }
  if (!/[0-9]/.test(password)) {
    errors.push("Password must include at least one number");
  }
  if (!/[!@#$%^&*]/.test(password)) {
    errors.push("Password must include at least one special character (!@#$%^&*)");
  }
  if (password === "100101") {
    errors.push("New password cannot be the default password");
  }

  return { valid: errors.length === 0, errors };
}
```
