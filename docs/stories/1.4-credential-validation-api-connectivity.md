# Story 1.4: Credential Validation and API Connectivity Test

## Status: Completed

## Description
As an admin, I want to validate my Meta API credentials before saving them, so that I know immediately if there's a configuration problem.

## Acceptance Criteria
- [x] Given I am on the Settings page with Meta API credentials filled in, when I click "Test Connection" button, then a loading spinner appears and button text changes to "Testing..."
- [x] Given I click "Test Connection", when the test is in progress, then the button is disabled to prevent duplicate requests
- [x] Given the test is successful, when the API responds, then I see a green success banner with: checkmark icon, "Connection successful", Meta account display name, phone number, and verification status
- [x] Given the test is successful, when the response is received, then the `last_tested_at` timestamp is updated in the database
- [x] Given the test fails with invalid credentials, when the API responds with 401, then I see a red error banner with: X icon, "Authentication failed", and message "Invalid access token or insufficient permissions"
- [x] Given the test fails with invalid Phone Number ID, when the API responds with 404, then I see error message: "Phone Number ID not found. Please verify the ID is correct and associated with your WABA."
- [x] Given the test fails with network error, when the request times out or fails, then I see error message: "Unable to connect to Meta API. Check your internet connection and try again."
- [x] Given the test fails, when error occurs, then troubleshooting suggestions are displayed (e.g., "Verify your access token has 'business_management' and 'whatsapp_business_messaging' permissions")
- [x] Given credentials have been tested successfully, when I click "Save Credentials", then the credentials are saved without requiring another test
- [x] Given I update any credential field after a successful test, when I modify WABA ID, Phone Number ID, or Access Token, then the test status is reset and I must test again before saving
- [x] Given the test is successful, when I view the Settings page later, then I see "Last tested: 2 hours ago" (relative timestamp) with green checkmark icon

## Technical Notes

### Meta API Endpoint for Testing
- **Endpoint**: `GET https://graph.facebook.com/v18.0/{phone_number_id}`
- **Headers**: `Authorization: Bearer {access_token}`
- **Success Response (200)**:
  ```json
  {
    "id": "123456789012345",
    "display_phone_number": "+55 11 3000-0000",
    "verified_name": "Cartão de Todos",
    "quality_rating": "GREEN",
    "code_verification_status": "VERIFIED",
    "name_status": "APPROVED"
  }
  ```
- **Error Responses**:
  - 401: Invalid or expired access token
  - 404: Phone Number ID not found or not associated with access token
  - 403: Insufficient permissions (missing required scopes)
  - 429: Rate limit exceeded
  - 500: Meta API internal error

### API Endpoints
- **POST /api/settings/credentials/test**
  - Headers: `Authorization: Bearer {token}`
  - Request body: `{ waba_id: string, phone_number_id: string, access_token: string }`
  - Response (success): `{ success: boolean, data: { display_name: string, phone_number: string, verified: boolean, quality_rating: string } }`
  - Response (error): `{ success: boolean, error: string, suggestions?: string[] }`
  - Status codes: 200 (test completed, check success field), 400 (validation error), 401 (unauthorized user), 500 (server error)

- **PUT /api/settings/credentials** (update from Story 1.3):
  - Add logic to check if credentials have been tested before saving
  - Update `last_tested_at` timestamp when test succeeds

### Backend Services
- **Meta API Adapter** (`backend/src/services/metaApiService.ts`):
  - Method: `testConnection(phoneNumberId: string, accessToken: string)`
  - Makes GET request to Meta API
  - Parses response and extracts display_name, phone_number, verified status
  - Handles all error cases (401, 404, 403, 429, 500, network errors)
  - Returns standardized result object: `{ success: boolean, data?: object, error?: string }`

- **Credential Service** (`backend/src/services/credentialService.ts`):
  - Method: `testCredentials(wabaId, phoneNumberId, accessToken)`
  - Calls Meta API adapter
  - Updates `last_tested_at` timestamp in database if test succeeds
  - Returns formatted response for frontend

### Error Handling and Troubleshooting Suggestions

#### 401 Unauthorized
- **Error Message**: "Authentication failed. Invalid access token or insufficient permissions."
- **Suggestions**:
  - "Verify your access token is active and not expired."
  - "Ensure the token has 'business_management' and 'whatsapp_business_messaging' permissions."
  - "Check that the token is associated with the correct Business Manager account."

#### 404 Not Found
- **Error Message**: "Phone Number ID not found."
- **Suggestions**:
  - "Verify the Phone Number ID is correct (15-17 digits)."
  - "Ensure the phone number is registered in your WhatsApp Business Account."
  - "Check that the access token has access to this phone number."

#### 403 Forbidden
- **Error Message**: "Insufficient permissions."
- **Suggestions**:
  - "Your access token is missing required permissions."
  - "Required scopes: business_management, whatsapp_business_messaging."
  - "Regenerate your System User token with the correct permissions in Meta Business Manager."

#### 429 Rate Limit
- **Error Message**: "Rate limit exceeded. Please try again in a few minutes."
- **Suggestions**:
  - "Meta API has rate limits (5000 requests/hour)."
  - "Wait 5-10 minutes before testing again."

#### Network Error / Timeout
- **Error Message**: "Unable to connect to Meta API."
- **Suggestions**:
  - "Check your internet connection."
  - "Verify Meta API is not experiencing downtime (status.facebook.com)."
  - "Try again in a few minutes."

- Test with access token that has expired (Meta returns 401)
- Test with access token that has insufficient permissions (Meta returns 403)
- Test with phone number ID that belongs to a different WABA (Meta returns 404)
- Test connection multiple times in quick succession (ensure no race conditions)
- Test connection while another admin is testing simultaneously (concurrent requests)
- Test with very long response time from Meta API (timeout after 10 seconds)
- Test result banner dismissal and re-testing (state management)

- Test connection does NOT save credentials automatically - admin must explicitly click "Save Credentials" after successful test.
- Troubleshooting suggestions are critical for non-technical admins (provide actionable guidance).
- Rate limiting on test endpoint prevents abuse (e.g., max 10 tests per minute per user, tracked in Redis).
- last_tested_at timestamp provides transparency and confidence that credentials are working.
- Consider caching successful test results for 5-10 minutes to reduce unnecessary Meta API calls (Phase 2 optimization).
- Access token is never logged or exposed in error messages (security requirement).
- Meta API version v18.0 is used (align with architecture document).
